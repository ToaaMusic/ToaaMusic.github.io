## Lua 本身

如果你有其他语言的基础，我认为学会写标准 Lua 脚本只需要一天

关于这门小巧且独特的语言，我对它的印象会用以下几个短语概括：

- 面向**表**的编程语言
- 有函数的 json
- 极其精简的 python
- 和 C 高度可互操作，为嵌入而生
（因此任何一种可调用 C API 的语言都可以成为 Lua 的宿主语言并实现互操作）

它有以下用处：

- 嵌入游戏做 mod
- 给应用提供灵活的配置系统
- 任何需要用户自定义逻辑并同步修改宿主语言内存的地方

## 万物皆为表 (table)

你不会在原生 lua 里见到任何除表以外的数据结构，但它们都可以用表来模拟。

Lua 中的表可以理解为一个键值对集合，写法类比 json：

```json
"my_table": {
  "name": "表",
  "a": 1
}
```

```lua
my_table = {
  name = "表"
  a = 1
}
```

这段代码就声明了一个变量，叫 “my_table”，类型为 table，你可以通过 "." 拿到表的内容

```lua
print(my_table.a)
```

这只是常规用法，实际上表里的值可以为任意类型，包括函数，甚至键也可以为任意类型，包括函数！
